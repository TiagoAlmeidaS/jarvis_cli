## Objetivo

Ter testes de integração que validem o fluxo do daemon usando apenas Google (Gemini / free tier), desde a subida até a execução de pelo menos um pipeline.

## Descrição

Implementar testes que cubram: (1) daemon sobe com só GOOGLE_API_KEY; (2) pipeline com provider: "google" e model: "gemini-2.0-flash" é executado; (3) fluxo não quebra sem OpenRouter.

## Implementação / Escopo

- **Cenário subida:** teste que inicia o daemon com GOOGLE_API_KEY, verifica scheduler ativo.
- **Cenário pipeline Google:** registra pipeline google, dispara execução, verifica chamada LLM e resposta (pipeline leve ou mock).
- **Onde:** jarvis-rs/daemon/tests/ (ex.: integration_google.rs) ou runbook de teste manual em docs.

## Critério de aceite

- Existe teste de integração ou runbook documentado que valida fluxo daemon + Google.
- Reproduzível (ex.: `cargo test -p jarvis-daemon --test integration_google`).
- Nenhum regresso com OpenRouter.
